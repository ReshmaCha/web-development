&

Ampersad (&-teken) is handig bij het nesten om tijdsverspilling te voorkomen

& verwijst altijd naar de bovenliggende selector

Je zou het kunnen zien als iets wat verwijderd en weer vervangen wordt

Bijv.

.parent {

.child {}

}

compileert naar

.parent .child {}

In Sass wordt dit gecompileerd middels een &-teken

.parent { & .child {} }

Dit is een voorbeeld van een parent
en dit is het voorbeeld van een child-selector
